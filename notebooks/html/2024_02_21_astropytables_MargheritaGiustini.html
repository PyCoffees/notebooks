0. Why using Astropy¶

I spent most of my life using software such as fv in order to manipulate flexible image transport system (FITS) files. I have always been quite slow, clumsy.

fv is the interactive FITS file viewer and editor, first developed at NASA Goddard in the 1990s as part of the FTOOLS.

Life is much better since I am using astropy.

The Astropy Project astropy is an open-source, community-developed core package for astronomy in Python, about 20-30 years younger than the FTOOLS.

Why? Because I can quickly:

  • rename columns;
  • perform numpy operations on and between cells or columns;
  • manually change cells values;
  • visualize data with matplotlib;
  • create new tables by filtering the original one using powerful masks;
  • merge tables in smart ways;
  • etc, etc

Let's work with data of the galaxy RX J1301.9+2747, one of the six (+2) known sources of X-ray quasi-periodic eruptions (QPEs).

QPEs are a new, peculiar cosmic signal associated with massive black holes in galactic nuclei, first discovered at CAB at the end of 2018/beginning of 2019 by Giovanni Miniutti, who was looking at the galaxy GSN 069 with XMM-Newton.

The data used in this tutorial are X-ray photons detected betwen $0.2-2$ keV by the EPIC-pn detector of the ESA satellite XMM-Newton in June 2022, while it has been pointing at RX J1301.9+2747 for about 33 hours.

XMM-Newton can actually detect photons up to 10 keV, but the signal of the QPE sources rapidly fades away at energies $> 2$ keV, thus at those energies the signal-to-noise ratio of our observations is negligible.

The data have been processed using software and calibration files provided by the XMM-Newton Science Operation Center, and from them a light curve (time series: rate of arrival of X-ray counts in the detector as a function of time) has been created and stored in the FITS file rxj_2022a_b250.lc.

2. Playing tables with Astropy¶

2.0 Let's open the FITS table with the Bayesian analysis results¶

In this basic example we will load a FITS file containing tabular data of five XMM-Newton observations of RX J1301.9+2747.

The data are the results of the Bayesian comparison between the $0.2-2$ keV light curves of the five observations and a model comprising a constant plus Gaussian emission lines.

The constant represents the quiescent X-ray emission of the galaxy, while the Gaussians represent quasi-periodic eruptions (QPEs).

2.1 Creating a new (smaller) table to work with¶

There are 32 QPEs in this table, and lots of data that we are not going to use for this brief tutorial.

Let's make a table retaining only the quantities that are of interest here: the QPE peak position in time, the QPE width, the QPE peak count rate, and the quiescence count rate; along with their 1$\sigma$ errors.

Each QPE has also an associated 'name', these are the non-consecutive integers, stored in the qpe_index array:
